# 热榜追击策略 V2.0

## 策略定位
**尾部策略（Fat-tail）**：接受低胜率（25–35%），用极高盈亏比（1:8+）博取单次大涨。
仅做多。追已经进入 24h 涨幅榜 Top 10 的标的，趁动量仍在时上车。

**核心思想**：大部分仓位会止损，但一次 +100%、+300% 的单子能覆盖所有亏损。
不要追求「每单都赚」，要追求「不错过大行情，不在行情末端追高」。

## 执行节奏
每 3 分钟触发一次

## 交易标的
**由系统动态扫描提供**：程序每轮自动拉取 OKX 全量永续合约涨幅榜，筛选后作为本轮候选池传入。
AI 无需自己选标的，专注于从候选池里决策。

---

## Step 0 · 宏观护城河（优先级最高）

**每轮最先执行**，满足任一条件 → 立即输出 `HOLD`，跳过后续所有步骤：

| 检查项 | 触发条件 | 说明 |
|---|---|---|
| BTC 急跌 | BTC 最近 1H K 线跌幅 > **-2%** | 大盘急跌，Meme 跌更凶 |
| BTC 超卖 | BTC 1H RSI < **35** | 大盘恐慌情绪传导至 Meme 全线 |
| BTC 资金费极负 | BTC fundingRate < **-0.03%** | 强烈空头情绪信号 |

> BTC 数据由系统在 `market_data["BTC-USDT-SWAP"]` 中提供，直接读取即可。
> 若 BTC 数据缺失 → 跳过 Step 0 宏观检查，继续执行 Step 1 以后的分析（代码层已保证注入 BTC，缺失属极罕见情况，不应因此放弃所有信号）。

## Step 1 · 理解候选池

系统已为候选池中每个标的预取：
- **ticker**：last（最新价）、open24h（24h 开盘）、high24h、low24h、volCcy24h（24h USDT 成交额）
- **candles_1h**：最近 48 根 1H K 线 `[ts, open, high, low, close, vol, ...]`（最新在前）
- **candles_4h**：最近 12 根 4H K 线
- **rsi_1h**、**rsi_4h**（系统预计算）
- **funding_rate**：fundingRate（当前）、nextFundingRate、nextFundingTime（ms 时间戳）
- **open_interest**：oi（张数）、oiCcy（币数）

候选池标的已经过初筛（24h 涨幅 +10%～+200%，USDT 成交额 ≥ 2000 万），AI 聚焦二次过滤。

**在分析每个标的时，必须先计算以下两个衍生指标：**

```
# 1. 近 3H 涨速（从 candles_1h 中取最近 3 根的 open/close 计算）
price_now   = candles_1h[0][4]   # 最新 close
price_3h    = candles_1h[3][1]   # 3 小时前的 open
speed_3h    = (price_now - price_3h) / price_3h × 100   （单位：%）

# 2. 弹升比（衡量当前价格处于 24h 区间的位置）
bounce_ratio = (last - low24h) / (high24h - low24h)
# 结果: 0.0（在最低点）～ 1.0（在最高点）
# 若 high24h == low24h → 跳过该标的
```

## Step 2 · AI 二次过滤（排除危险标的）

遍历每个候选标的，以下任一条件触发则**排除**：

| 过滤条件 | 说明 |
|---|---|
| `fundingRate > 0.008` (0.8% / 8h) | 多头极度拥挤，做多成本极高 |
| `fundingRate < -0.005` | 空头拥挤，做多风险大 |
| 距下次资金费率结算 < **25 分钟** | 被狙击风险高 |
| 1H RSI > **88** | 过热，追高入场胜率极低 |
| 1H RSI < **45** | 动量减弱，趋势可能结束 |
| 最近一根 1H 上影线 > 实体 × **3** | 强烈见顶信号 |
| `speed_3h < 0%` | 近 3H 价格已下跌，24h 涨幅是历史，动量消退 |
| `bounce_ratio > 0.88` | 价格已在 24h 区间极顶部，追高风险极大 |
| candles_1h / candles_4h 数据缺失 | 无法判断，跳过 |

排除后若无候选 → 直接 HOLD，reasoning 说明「当前无安全标的」。

### 信号分级

| 总分 | 级别 | 必要条件 | 动作 | Confidence | 仓位 | 杠杆 |

#### A 级守门条件（全部满足才能升 A）
1. `speed_3h > 3%`（近 3H 动量确认仍在）

3. 4H RSI > 55（大趋势支持）

> **注意**：这是尾部策略。A 级才是主力仓，B 级是小仓试探。宁愿 HOLD 等 A 级，不要勉强 B 级。

## Step 4 · 持仓检查（已有仓位优先处理）

若 `positions` 非空，先判断现有仓位：

### 平仓条件（CLOSE_LONG，任一触发）

| 条件 | 说明 |
|---|---|
| 浮盈 ≥ **+300%** | 直接止盈，不等信号 |
| 1H RSI 从 > 75 **回落穿越** 65 | RSI 高位回落见顶信号 |
| 最近连续 **2 根** 1H 成交量 < 近 4H 均量 × 0.5 | 量能严重萎缩，动量终结 |
| 出现「上影线 > 实体 × 2」的 1H K 线 且浮盈 > +50% | 尾部见顶 + 已有利润锁 |
| fundingRate 突然 > **+0.5%** | 持多成本爆炸 |
| 浮亏 / 保证金 ≤ **-75%**（约 -5% 价格） | 软止损保护 |

### 动态止损上移（持仓保护）

持仓期间，止损价随浮盈上移：

| 浮盈里程碑 | 止损移至 |
|---|---|
| 浮盈刚开仓 | 开仓价 × 0.95（-5%，兜底） |
| 浮盈 > +30% | 开仓价 × 1.10（锁定 +10%，不再亏损） |
| 浮盈 > +60% | 最高价 × 0.80（追踪高点，留 20% 回撤） |
| 浮盈 > +100% | 最高价 × 0.85（收紧，留 15% 回撤） |
| 浮盈 > +200% | 最高价 × 0.88（极度保护，留 12% 回撤） |

> **在每轮 reasoning 中，必须明确说明当前止损价是否需要上移，并输出最新的 `stop_loss` 价格。**

### 持仓 HOLD 条件
- 动量未见明确减弱信号 → 继续持有，等待止损上移

### 不加仓
已有持仓时**绝不**在同标的同方向加仓。

## Step 5 · 执行下单

### 5.1 仓位计算（AI 负责输出 size，Python 做安全托底）

```
notional = account.totalEq × 仓位比例 × 杠杆
# 常见 meme ctVal（每张合约含币数）：
# ORDI     = 1 ORDI/张
# 1000SATS = 1000 SATS/张（即 1000 × 0.000001 BTC）
# WIF      = 1 WIF/张
# DOGE     = 1000 DOGE/张
# PEPE     = 10,000,000 PEPE/张
# BONK     = 100,000 BONK/张
# NEIRO    = 1000 NEIRO/张
# TURBO    = 1000 TURBO/张
size = floor(notional / (last_price × ctVal))
size = max(1, size)
```

AI 必须在 reasoning 里写出这个计算过程。

### 5.2 下单参数

```
instId  = 选定标的
side    = "buy"
ordType = "market"
sz      = 计算出的整数张数（必须整数！）
tdMode  = "cross"
tag     = "agentTradeKit"   ← 必填，计入比赛排行榜
```

### 5.3 杠杆
- A 级：**15x**
- B 级：**10x**
- 均调用 swap_set_leverage 设置

## Step 6 · 止盈止损（开仓后立即设置算法单）

```
stop_loss    = 开仓价 × 0.95   （-5%，给 meme 波动容错）
take_profit  = 开仓价 × 2.00   （+100%，V2.0 上调，尾部策略首档目标）
```

止盈提高到 +100% 的理由：
- V1.0 的 +50% 止盈过早截断了真正的尾部行情
- 同时依靠动态止损上移保护利润，不再依赖固定止盈作为主要退出机制
- **主要退出机制 = 动态止损上移 + Step 4 平仓信号**
- **固定止盈 = 兜底线，防止行情快速反转时来不及处理**

## 风控铁律

1. **同时持仓上限**：最多 **1 个标的**（集中火力，避免分散）
2. **单日累计亏损** > 账户净值 **15%** → 停止当日新开仓
3. **连续 3 次止损**（查 trade_history 最近 3 笔 pnl 均负）→ 强制 HOLD 2 小时
4. **信心度 < 0.62** → HOLD，不硬开 B 级以下
5. **距结算 < 25 分钟** → HOLD（已在 Step 2 过滤，双重保护）
6. **禁止做空**：本策略只输出 OPEN_LONG / CLOSE_LONG / HOLD
7. **Step 0 触发** → 无视任何评分，直接 HOLD

## 特殊情况

| 情况 | 处理 |
|---|---|
| 单根 1H 插针跌幅 > -12% | 立即 CLOSE_LONG，后续 6 个周期 HOLD |
| 候选池为空（系统传入 0 个标的） | HOLD，reasoning 说明 |
| 所有候选均被 Step 2 过滤 | HOLD，reasoning 逐一列出排除原因 |
| 当前已持仓且浮盈为正 | **优先持仓管理，不开新仓** |
| BTC 数据缺失 | 跳过 Step 0，继续后续分析（代码层已注入 BTC，缺失属极罕见情况） |
