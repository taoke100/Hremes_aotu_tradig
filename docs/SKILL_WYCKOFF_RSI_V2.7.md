# 威科夫 RSI 稳健生存版 V2.7
# Wyckoff RSI Survivor V2.7

## 策略定位
专为低资金、高生存率设计的**反转捕捉策略**。
以 1H 级别 RSI(6) 极端区为入场铁律，结合威科夫量价背离与结构判断进行 AI 二次确认，统一使用超低杠杆（3x），单笔账户风险锁定在 0.25% 以内，彻底杜绝爆仓风险。

## 执行节奏
每 15 分钟触发一次

## 交易标的
- 主标的：BTC-USDT-SWAP、ETH-USDT-SWAP、SOL-USDT-SWAP
- 每轮 AI 在三个标的中选出信号最强的一个执行，或 HOLD

---

## Step 1 · 行情数据采集

调用 `market_get_candles` 分别获取 BTC / ETH / SOL 永续合约的：
- **1H K 线（最近 24 根）**：用于计算 RSI(6)、识别价格结构（Spring/Upthrust）、量价背离
- **4H K 线（最近 12 根）**：用于判断更大周期趋势方向，过滤逆势信号

调用 `market_get_funding_rate` 获取三个标的当前及预测资金费率。
调用 `market_get_open_interest` 获取三个标的持仓量变化趋势。

---

## Step 2 · RSI 极端区扫描（入场铁律）

计算每个标的的 **1H RSI(6)**（使用最近 24 根 1H K 线）。

**铁律**：1H RSI(6) 未触及极端区，直接跳过该标的。

| 方向 | RSI 条件 | 含义 |
|---|---|---|
| 做多候选 | RSI(6) < **20** | 短期极度超卖，潜在反弹 |
| 做空候选 | RSI(6) > **80** | 短期极度超买，潜在回调 |

> RSI 铁律是**充分必要条件**：未触及极端区 → 不得开仓，无论其他信号多强。

---

## Step 3 · AI 综合判断（核心推理）

对通过 Step 2 的候选标的，AI 必须逐项推理以下 4 个维度，综合评级后给出决策。

### 3.1 威科夫结构识别（最高价值信号）

在最近 12 根 1H K 线中，AI 须识别以下结构：

**做多结构（Spring）**：
- 价格短暂跌破近期支撑区后迅速收回（假跌破）
- 该跌破 K 线成交量明显缩量（恐慌性抛盘但无承接者）
- 随后一根 K 线强势收复，确认反弹

**做空结构（Upthrust / UT）**：
- 价格短暂突破近期阻力区后迅速回落（假突破）
- 该突破 K 线成交量明显缩量或不放大（无真实买盘）
- 随后一根 K 线快速跌回，确认回落

| 结构识别 | 信号等级 |
|---|---|
| 明确识别到 Spring / Upthrust | **A 级** |
| RSI 极端但结构不明确 | **B 级** |

### 3.2 量价背离验证

AI 须判断以下背离（任一满足则加权）：

**顶背离（配合做空）**：
- 价格创新高，但 RSI(6) 未创新高（RSI 顶背离）
- 价格创新高，但成交量萎缩（量价背离）

**底背离（配合做多）**：
- 价格创新低，但 RSI(6) 未创新低（RSI 底背离）
- 价格创新低，但成交量萎缩（恐慌盘减少）

### 3.3 4H 趋势过滤

| 4H 趋势状态 | 做多信号处理 | 做空信号处理 |
|---|---|---|
| 4H 趋势向上（最近 3 根 4H 为阳线，或 4H RSI > 55） | 加权：趋势顺向 ✅ | 降格：逆势，B→需再确认 |
| 4H 趋势向下（最近 3 根 4H 为阴线，或 4H RSI < 45） | 降格：逆势，B→需再确认 | 加权：趋势顺向 ✅ |
| 4H 趋势中性（RSI 45-55） | 中性，权重不变 | 中性，权重不变 |

### 3.4 资金费率与持仓量检查

| 检查项 | 条件 | 处理 |
|---|---|---|
| 做多：资金费率偏正 | fundingRate > 0.05% | 仓位减半（多头拥挤，轧多风险） |
| 做空：资金费率偏负 | fundingRate < -0.05% | 仓位减半（空头拥挤，轧空风险） |
| 极端费率（任意方向） | \|fundingRate\| > 0.1% | 跳过本标的，HOLD |
| OI 同向增加 | OI 与开仓方向同向扩张 | 确认加权 ✅（新资金入场） |
| 距结算 < 30 分钟 | nextFundingTime 距今 < 30min | 本轮 HOLD，等结算后再判断 |

### 3.5 AI 综合评级

综合以上 4 个维度，评出最终信号等级：

| 等级 | 条件 | 动作 | Confidence |
|---|---|---|---|
| **A 级** | RSI 铁律触发 + 识别到 Spring/UT + 任一背离 + 4H 顺向 | OPEN_LONG / OPEN_SHORT | 0.75 – 0.95 |
| **B 级** | RSI 铁律触发 + 背离或结构确认（满足其一）| OPEN_LONG / OPEN_SHORT | 0.55 – 0.74 |
| **跳过** | RSI 铁律触发但无任何结构/背离确认 | HOLD | < 0.55 |

> AI 须在 reasoning 中逐项写明：RSI 数值、识别到的威科夫结构、背离类型、4H 趋势状态、资金费率值，最终给出评级依据。

## Step 4 · 仓位计算与执行下单

### 4.1 仓位计算

```
杠杆 = 3（固定，不可更改）
保证金占比：
  A 级信号：账户净值 × 5%（名义价值 = 净值 × 5% × 3 = 净值 × 15%）
  B 级信号：账户净值 × 3%（名义价值 = 净值 × 3% × 3 = 净值 × 9%）

# 合约面值参考（ctVal）：
# BTC-USDT-SWAP：0.01 BTC / 张
# ETH-USDT-SWAP：0.1  ETH / 张
# SOL-USDT-SWAP：1    SOL / 张

size = floor( 账户净值 × 保证金占比 × 杠杆 / (当前价格 × ctVal) )
size = max(1, size)   # 最小 1 张
```

AI 必须在 reasoning 中写出完整计算过程。

### 4.2 下单参数

调用 `swap_place_order`：

```
instId  = 选定的标的（BTC/ETH/SOL-USDT-SWAP）
side    = "buy"（做多）或 "sell"（做空）
ordType = "market"
sz      = 计算出的整数张数（必须为正整数）
tdMode  = "cross"
tag     = "agentTradeKit"   ← 必填，否则不计入比赛排行榜
```

### 4.3 杠杆设置

开仓前调用 `swap_set_leverage`：
```
lever   = 3
mgnMode = "cross"
```

## Step 5 · 止损与动态追踪设置

开仓后立即调用 `swap_place_algo_order` 设置初始止损：

### 5.1 初始止损（Hard SL）

```
多头初始 SL = 开仓价 × 0.95   （-5%，给予充分呼吸空间）
空头初始 SL = 开仓价 × 1.05   （+5%）
```

### 5.2 保本触发

每轮持仓检查中，AI 须判断：
- 当浮盈 ≥ **+2%**（价格方向移动 2%）时 → 将止损更新至**成本价（开仓价）**
- 调用 `swap_place_algo_order` 修改止损价

### 5.3 移动追踪（3% / 1% 步进）

价格每向有利方向移动 **3%**，止损同步推进 **1%**：

| 价格有利移动幅度 | 止损移至（多头示例，以开仓价为基准） |
|---|---|
| 开仓时 | 开仓价 × 0.95 |
| 盈利 +2% | 开仓价 × 1.00（保本） |
| 盈利 +5% | 开仓价 × 1.01 |
| 盈利 +8% | 开仓价 × 1.02 |
| 盈利 +11% | 开仓价 × 1.03 |
| … | 每 +3% 追一次，止损跟进 +1% |

> AI 每轮须在 reasoning 中说明：当前浮盈百分比、止损是否需要更新、新的止损价格。

### 5.4 止盈设置

初始止盈设置（与开仓同时下单）：
```
多头初始 TP = 开仓价 × 1.10   （+10%，设置算法止盈，赚到后追踪止损继续持有）
空头初始 TP = 开仓价 × 0.90   （-10%）
```

主要退出机制为**追踪止损**，固定止盈为兜底线。

## Step 6 · 持仓管理

每轮触发时，若已有持仓，**优先处理持仓**，再决定是否开新仓。

### 持仓评估

| 情况 | 处理 |
|---|---|
| 追踪止损已触发 | 交易所自动平仓，本轮寻找新信号 |
| 浮盈 ≥ +2%，止损未保本 | 立即更新止损至成本价 |
| 持仓标的 RSI 反向穿越 50 | 提示可能趋势结束，考虑手动干预（AI 输出警告信息） |
| 已有持仓，当前有新信号 | 不加仓，不双开，优先管理现有仓位 |

## 风控铁律

1. **杠杆上限**：统一 **3x**，任何情况不得提高
2. **单笔风险**：账户净值 × 保证金占比 × 5%（止损空间）= 约 **0.25% 账户净值**（A级）
3. **日度回撤熔断**：当日净值回撤超 **5%** → 停止一切新开仓，只允许持仓管理或平仓
4. **同时持仓上限**：最多 **1 个标的**，禁止同时多空对冲
5. **连续止损熔断**：最近 3 笔均为亏损 → 强制暂停 4 小时，不开新仓
6. **无信号纪律**：未触及 RSI 铁律极端区 → 坚决 HOLD，不勉强入场
7. **资金费狙击防护**：距结算 < 30 分钟 → HOLD（已在 Step 3.4 过滤，双重保护）
8. **tag 合规**：每笔下单必须携带 `tag = "agentTradeKit"` 否则不计入排行榜
