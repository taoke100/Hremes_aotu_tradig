# 山寨币流动性枯竭做空策略 V1.0
# Altcoin Liquidity Exhaustion Short Strategy

## 策略定位
**均值回归策略（Mean-reversion Short）**：捕捉山寨币暴涨后的流动性枯竭拐点，做空回撤。
仅做空。专找 24h 涨幅榜上**已经涨完、量能衰退、多头疲惫**的标的，在拐点入场做空。

**核心思想**：山寨币暴涨 +50%～+200% 后，90% 会在 48 小时内回撤 30%～60%。
不追涨、不猜底，只做「涨完了但还没跌」的那个窗口。

## 执行节奏
每 3 分钟触发一次

## 交易标的
**由系统动态扫描提供**：程序每轮自动拉取 OKX 全量永续合约涨幅榜，筛选后作为本轮候选池传入。
AI 无需自己选标的，专注从候选池里寻找「流动性枯竭」信号。

---

## Step 1 · 理解候选池

系统已为候选池中每个标的预取：
- **ticker**：last（最新价）、open24h（24h 开盘）、high24h、low24h、volCcy24h（24h USDT 成交额）
- **candles_1h**：最近 24 根 1H K 线 `[ts, open, high, low, close, vol, ...]`（最新在前）
- **candles_4h**：最近 12 根 4H K 线
- **rsi_1h**、**rsi_4h**（系统预计算，基于 RSI(14)）
- **funding_rate**：fundingRate（当前）、nextFundingRate、nextFundingTime（ms 时间戳）
- **open_interest**：oi（张数）、oiCcy（币数）

候选池标的已过初筛（24h 涨幅 +10%～+200%，USDT 成交额 ≥ 2000 万）。

**AI 在分析每个标的时，必须先计算以下衍生指标：**

```
# 1. 距高点回撤比（从 24h 最高价回落了多少）
drawdown = (high24h - last) / high24h × 100   （单位：%）
# 结果: 0% = 仍在最高点；15% = 已从高点回落 15%

# 2. 量能萎缩比（最新 1H 成交量 vs 最近 6H 均值）
vol_ratio = candles_1h[0].vol / mean(candles_1h[1:7].vol)
# 结果: < 0.5 = 量能严重萎缩；< 0.3 = 极度枯竭

# 3. 上影线比率（最新 1H K 线）
upper_shadow = high - max(open, close)
body = abs(close - open)
shadow_ratio = upper_shadow / body   （body=0 时视为无效 K 线跳过）
```

## Step 2 · AI 二次过滤（排除不适合做空的标的）

遍历每个候选标的，以下任一条件触发则**排除**：

| 过滤条件 | 说明 |
|---|---|
| `fundingRate < -0.005` (-0.5% / 8h) | 空头已极度拥挤，做空成本高 + 轧空风险 |
| 距下次资金费率结算 < **25 分钟** | 结算前波动大，被狙击风险高 |
| 1H RSI < **40** | 已经跌了很多，做空抢在别人后面了 |
| `drawdown > 25%` | 已从高点大幅回落，做空空间不足 |
| `vol_ratio > 1.5` | 量能仍在放大，趋势仍强，不宜逆势做空 |
| 最近 3 根 1H 全是阳线且每根涨幅 > 1% | 强势连涨中，动量未减，不做空 |
| 24h 涨幅 < **+20%** | 涨幅太小，做空空间有限 |
| candles_1h / candles_4h 数据缺失 | 无法判断，跳过 |

排除后若无候选 → 直接 HOLD，reasoning 说明"当前无适合做空的标的"。

对通过过滤的候选标的逐一打分，选**总分最高**的一个。

| 代号 | 条件 | 分值 |

| | `vol_ratio ∈ [0.3, 0.5]`（明显萎缩）| +2 |
| | `vol_ratio ∈ [0.5, 0.7]`（轻度萎缩）| +1 |
| **S3 连阴确认** | 最近 3 根 1H 全是阴线（close < open） | +3 |
| | 最近 3 根中有 2 根阴线 | +1 |
| **S4 RSI 高位回落** | 1H RSI 在 65～78 区间（从超买回落中）| +2 |

| **S5 上影线见顶** | 最近 3 根 1H 中任一出现 `shadow_ratio > 2` | +2 |

| **S8 OI 下降** | OI 与价格同步下降（多头平仓出逃） | +1 |

### 信号分级

| 总分 | 级别 | 必要条件 | 动作 | Confidence | 仓位 | 杠杆 |
|---|---|---|---|---|---|---|
| ≥ 14 且 S2+S3 全满足 | **A 级** | S2≥+2 且 S3≥+1 且 S6≥+1 | OPEN_SHORT | 0.80–0.92 | 净值 × **12%** | **10x** |
| 9–13 且 S2≥+1 | **B 级** | S2≥+1 且 (S3≥+1 或 S5≥+1) | OPEN_SHORT | 0.62–0.79 | 净值 × **6%** | **8x** |

> **注意**：做空比做多风险大（山寨币可能二次暴拉），A 级才是主力仓，B 级试探仓。
> 做空杠杆上限 10x（低于做多策略的 15x），控制风险。

## Step 4 · 持仓检查（已有仓位优先处理）

若 `positions` 非空，先判断现有仓位：

### 平仓条件（CLOSE_SHORT，任一触发）

| 条件 | 说明 |
|---|---|
| 浮盈 / 保证金 ≥ **+100%** 且最近 1H 转阳 | 主动止盈锁利 |
| 浮盈 ≥ **+200%** | 直接止盈，不等信号 |
| fundingRate 突然 < **-0.5%** | 空头成本爆炸 |
| 浮亏 / 保证金 ≤ **-60%**（约 +6% 价格反弹） | 软止损保护 |
| 最近 3 根 1H 全为强阳（每根涨幅 > 1%）| 趋势反转信号，立即止损 |
| `vol_ratio > 2.0`（成交量突然爆发） | 新一轮拉升可能开始 |

### 持仓 HOLD 条件
- 浮盈 +10%～+100%，量能持续萎缩，趋势下行未破 → 继续持有

### 不加仓
已有持仓时**绝不**在同标的同方向加仓。

## Step 5 · 执行下单

### 5.1 仓位计算（AI 负责输出 size，Python 做安全托底）

```
notional = account.totalEq × 仓位比例 × 杠杆
# 常见 meme ctVal（每张合约含币数）：
# ORDI     = 1 ORDI/张
# 1000SATS = 1000 SATS/张
# WIF      = 1 WIF/张
# DOGE     = 1000 DOGE/张
# PEPE     = 10,000,000 PEPE/张
# BONK     = 100,000 BONK/张
# NEIRO    = 1000 NEIRO/张
# TURBO    = 1000 TURBO/张
size = floor(notional / (last_price × ctVal))
size = max(1, size)
```

AI 必须在 reasoning 里写出这个计算过程。

### 5.2 下单参数

```
instId  = 选定标的
side    = "sell"       ← 做空
ordType = "market"
sz      = 计算出的整数张数（必须整数！）
tdMode  = "cross"
tag     = "agentTradeKit"   ← 必填，计入比赛排行榜
```

### 5.3 杠杆
- A 级：**10x**
- B 级：**8x**
- 均调用 swap_set_leverage 设置

## Step 6 · 止盈止损（开仓后立即设置算法单）

```
stop_loss    = 开仓价 × 1.06   （+6%，做空止损在价格上方）
take_profit  = 开仓价 × 0.70   （-30%，目标标的回撤 30%）
```

**为什么止盈 -30%**：
山寨币暴涨 +80%～+200% 后的典型回撤是 30%～60%。
止盈 -30% 保守但合理，配合持仓管理中的平仓信号，大部分单子在 -15%～-30% 之间退出。

---

## 风控铁律

1. **同时持仓上限**：最多 **1 个标的**（集中火力，做空更需谨慎）
2. **单日累计亏损** > 账户净值 **12%** → 停止当日新开仓
3. **连续 3 次止损**（查 trade_history 最近 3 笔 pnl 均负）→ 强制 HOLD 3 小时
4. **信心度 < 0.62** → HOLD，不硬开 B 级以下
5. **距结算 < 25 分钟** → HOLD（已在 Step 2 过滤，双重保护）
6. **禁止做多**：本策略只输出 OPEN_SHORT / CLOSE_SHORT / HOLD
7. **止损空间上限**：任何情况止损不超过开仓价 +8%（防止极端亏损）

---

## 特殊情况

| 情况 | 处理 |
|---|---|
| 单根 1H 暴涨 > +15% | 立即 CLOSE_SHORT，后续 6 个周期 HOLD |
| 候选池为空（系统传入 0 个标的） | HOLD，reasoning 说明 |
| 所有候选均被 Step 2 过滤 | HOLD，reasoning 逐一列出排除原因 |
| 当前已持仓且浮盈为正 | **优先持仓管理，不开新仓** |
| 标的突然公布重大利好 / 上所消息 | AI 无法判断，由止损保护 |
